# 后台启动和日志管理指南

## 日志配置

已配置日志按天分割，日志文件存储在 `logs/` 目录：

- **app.log** - 当天的应用日志
- **app.log.2024-01-15** - 历史日志（按日期命名）
- 自动保留最近 30 天的日志

日志配置：
- 每天午夜自动分割
- 自动添加日期后缀
- UTF-8 编码
- 同时输出到控制台和文件

## 方案一：使用 systemd 服务（推荐）

### 1. 修改服务配置文件

编辑 `invoice-backend.service`，修改以下内容：

```ini
User=your_username                                    # 改为你的用户名
WorkingDirectory=/path/to/invoice-miniprogram/backend # 改为实际路径
Environment="PATH=/path/to/venv/bin"                  # 改为虚拟环境路径
ExecStart=/path/to/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
```

### 2. 安装服务

```bash
# 给脚本添加执行权限
chmod +x service.sh

# 安装服务
sudo ./service.sh install
```

### 3. 管理服务

```bash
# 启动服务
sudo systemctl start invoice-backend

# 停止服务
sudo systemctl stop invoice-backend

# 重启服务
sudo systemctl restart invoice-backend

# 查看状态
sudo systemctl status invoice-backend

# 查看日志
sudo journalctl -u invoice-backend -f

# 或使用脚本
sudo ./service.sh start
sudo ./service.sh stop
sudo ./service.sh restart
sudo ./service.sh status
sudo ./service.sh logs
```

### 4. 开机自启

服务安装后会自动设置开机自启，无需额外配置。

### 5. 卸载服务

```bash
sudo ./service.sh uninstall
```

## 方案二：使用 nohup（简单方案）

### 1. 启动服务

```bash
# 给脚本添加执行权限
chmod +x start.sh stop.sh

# 启动服务
./start.sh
```

服务会在后台运行，日志输出到：
- `logs/app.log` - 应用日志（按天分割）
- `logs/nohup.log` - 启动日志

### 2. 停止服务

```bash
./stop.sh
```

### 3. 查看日志

```bash
# 查看应用日志
tail -f logs/app.log

# 查看启动日志
tail -f logs/nohup.log

# 查看历史日志
ls logs/app.log.*
```

### 4. 查看进程

```bash
# 查看进程
ps aux | grep uvicorn

# 查看 PID
cat logs/app.pid
```

## 方案三：使用 screen（临时方案）

```bash
# 创建 screen 会话
screen -S invoice-backend

# 在 screen 中启动服务
uvicorn main:app --host 0.0.0.0 --port 8000

# 按 Ctrl+A 然后按 D 退出 screen（服务继续运行）

# 重新连接到 screen
screen -r invoice-backend

# 停止服务：重新连接后按 Ctrl+C
```

## 日志管理

### 查看实时日志

```bash
# 查看应用日志
tail -f logs/app.log

# 查看最近 100 行
tail -n 100 logs/app.log

# 查看特定日期的日志
cat logs/app.log.2024-01-15
```

### 搜索日志

```bash
# 搜索错误日志
grep "ERROR" logs/app.log

# 搜索特定用户的日志
grep "oRDIz3Z_X15V-MCtLtUkhjdBHiIg" logs/app.log

# 搜索所有日志文件
grep "ERROR" logs/app.log*
```

### 清理旧日志

日志会自动保留 30 天，超过 30 天的日志会被自动删除。

如果需要手动清理：

```bash
# 删除 30 天前的日志
find logs/ -name "app.log.*" -mtime +30 -delete

# 删除所有历史日志
rm logs/app.log.*
```

## 常见问题

### 1. 服务无法启动

检查：
- 端口 8000 是否被占用：`lsof -i :8000`
- Python 虚拟环境是否激活
- 依赖是否安装完整：`pip install -r requirements.txt`

### 2. 日志文件过大

- 检查日志配置是否正确
- 确认日志轮转是否正常工作
- 手动清理旧日志

### 3. 服务意外停止

查看日志：
```bash
# systemd 服务
sudo journalctl -u invoice-backend -n 100

# nohup 方式
tail -n 100 logs/nohup.log
tail -n 100 logs/app.log
```

### 4. 修改代码后重启服务

```bash
# systemd 方式
sudo systemctl restart invoice-backend

# nohup 方式
./stop.sh
./start.sh
```

## 性能监控

### 查看资源占用

```bash
# 查看进程资源占用
ps aux | grep uvicorn

# 实时监控
top -p $(cat logs/app.pid)

# 或使用 htop
htop -p $(cat logs/app.pid)
```

### 查看端口监听

```bash
# 查看端口状态
netstat -tlnp | grep 8000

# 或使用 ss
ss -tlnp | grep 8000
```

## 推荐配置

生产环境推荐使用 **systemd 服务**，因为：
- ✅ 开机自启
- ✅ 自动重启（崩溃后）
- ✅ 日志管理完善
- ✅ 资源限制
- ✅ 权限控制

开发/测试环境可以使用 **nohup** 或 **screen**，更加灵活。
